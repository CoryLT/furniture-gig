## What we BUILT this session (code, all pushed)
- `app/books/reconcile/page.tsx` — the **reconcile screen** (`/books/reconcile`).
  Shows the oldest unsorted bank line; you pick a category (money-out → expense
  accounts or Owner's Draws; money-in → income accounts or Owner's Contributions),
  pick the bank/asset account, optional piece + contact, then:
  - **Save & next** → calls `record_expense` (out) or `record_cash_sale` (in),
    sets `books_bank_feed.handled=true` + `transaction_id` = the new entry.
  - **Skip for now** → sets `status='skipped'` (handled stays false); queue orders
    `status asc` so fresh ('pending') come before 'skipped' → it comes back later.
  - **Not business** → `handled=true`, no entry.

  that reveals a name box. Used on reconcile + Log a sale + Log an expense for both
  the piece and the person fields. On save, the server action creates the
  `inventory_pieces` / `contacts` row (owner-stamped) and tags it.
