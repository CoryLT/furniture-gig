## GOTCHAS discovered this session (important)
- **Supabase dashboard was very flaky** all session: repeated
  "Failed to fetch (api.supabase.com)" errors. They're cosmetic/transient ‚Äî the
  query often ran anyway; verify with a count rather than trusting the error.
- **Stuck "Read Replica" source ‚Üí "Connection string is missing"** when running
  SQL. The SQL editor got stuck pointing at a non-existent read replica with no
  way to switch back to Primary in the dropdown. FIX: open Supabase in a fresh /
  incognito window ‚Äî a new session defaults to the Primary (writable) database.
- **Stray tables in the marketplace DB:** a pre-existing `inventory_items`
  (and/or `bank_feed`) exists WITHOUT an owner_user_id column and isn't used by
  the app. `create table if not exists` silently skips it, then index/policy on
  owner_user_id fails. That's why new tables must be `books_`-prefixed.
- **Copying a generated INSERT from a result cell adds wrapping double-quotes**
  (CSV style) that break the paste. Rebuild the INSERT clean, or strip the outer
  quotes. (Easiest: have Cory paste the generator output back, then hand him a
  clean ready-to-run version.)
